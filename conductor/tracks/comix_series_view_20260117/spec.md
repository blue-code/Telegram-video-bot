# 만화책 시리즈 뷰 전환 및 읽기 진행률 대시보드 구현

#### 1. 개요 (Overview)
- **목적**: 만화책 라이브러리를 시리즈별로 그룹화하여 관리 편의성을 높이고, 대시보드에서 최근 읽던 책을 바로 이어볼 수 있는 기능을 제공합니다.
- **주요 변경 사항**:
    - 만화책 메뉴의 기본 진입점을 '시리즈 목록'으로 변경.
    - 대시보드에 '최근 읽은 만화책' 섹션 추가.
    - 새로운 파일명 패턴 지원 및 기존 데이터 시리즈 마이그레이션 도구 개발.

#### 2. 기능 요구사항 (Functional Requirements)
- **UI/UX 개선**:
    - **시리즈 목록 뷰**: 만화책 메뉴 클릭 시 가장 먼저 표시되며, 각 시리즈는 대표 이미지와 함께 시리즈명, 포함된 권수 정보를 표시합니다.
    - **전체 파일 뷰**: 기존의 플랫한 목록은 탭이나 필터 옵션으로 유지합니다.
    - **대시보드 위젯**: 최근 읽은 만화책 3~5개를 썸네일, 제목, 시리즈명과 함께 표시하며 클릭 시 마지막 페이지로 바로 이동합니다.
- **데이터 관리**:
    - **시리즈 식별 로직 강화**:
        - `제목 N화.zip` (예: "그거 그렇게 하는거 아닌데 9화.zip")
        - `제목 - 01화.zip` (하이픈)
        - `[작가명] 제목 01.zip` (대괄호)
        - `제목 (01).zip` (괄호)
    - **마이그레이션 프로그램**:
        - DB 내 기존 만화책 레코드를 스캔하여 새로운 패턴에 맞춰 시리즈 정보를 업데이트합니다.
        - 실행 방식: 웹 관리자 버튼, 독립 실행 스크립트, 서버 시작 시 자동 실행 지원.

#### 3. 비기능 요구사항 (Non-Functional Requirements)
- **성능**: 시리즈 그룹화 쿼리 최적화로 대량의 파일이 있어도 로딩 속도를 유지합니다.
- **안정성**: 마이그레이션 과정에서 기존 데이터(파일 ID, 읽기 진행률 등)가 유실되지 않도록 보장합니다.

#### 4. 수락 기준 (Acceptance Criteria)
- [ ] 만화책 메뉴 접속 시 시리즈별로 묶인 화면이 먼저 나타나는가?
- [ ] 대시보드에서 최근 읽던 만화책의 '이어보기'가 정상 작동하는가?
- [ ] `제목 N화.zip` 패턴의 파일이 자동으로 시리즈로 인식되는가?
- [ ] 마이그레이션 스크립트 실행 후 기존 파일들이 시리즈로 올바르게 재분류되는가?
